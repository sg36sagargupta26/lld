Design a Rate Limiter in Spring Boot

there are two major ways to implement rate limiter in Spring boot, that is either by using handlerinterceptor or filter
Lets create a very simple and naive rate limiter we are using concurrent hashmap and we have created a scheduled task which clears entire map every minute

We set say for the details(say ip/customerID) as key to map and when the customer say hits our api we trigger it and from there we are using map.merge to get current counts
and if its more than certain limit we throw 429 status too many requests

There are small problems with the implementation:

Q>What happens at 59th second vs 1st second of next minute?
Q>Can a user send 2× limit in ~2 seconds?

yeah in the current implementation it clears all the details every minute, so instead of Map<userid, count>
we have now map<userId, List<TimeStamps>> where we check if the size is greater than allowed requests
also we have updated the scheduled task to remove users if they are not a part of active time region

Q>What happens under: 1000 concurrent requests for same user and other atomicity concerns
A> we were missing a syncronised block for timestamps array

Q>Why are you using a copy-on-write structure…when you are already locking it?
thats correct we don't need sycronised block on copy on write list, so I have replaced with arrayDeque

Q>Multi-instance consistency
To ensure multi instance consistency instead of using any data structure I am using database table to store user id, and time since now we have decoupled
the storage area and code this will ensure multi instance consistency
added and index for faster look up and a cron job every hour to clear the details
